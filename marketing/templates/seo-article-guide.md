# SEO記事ガイド：おくすり味ナビ

---

## 絶対守ること（全記事共通）

医薬品情報の記事は誤りが健康被害に直結する。以下は全記事で必ず守ること。

- [ ] おすすめ/NGの食品情報は、**必ずアプリのデータ（= 国立成育医療研究センター資料）と照合**してから書く
- [ ] 「この食品で飲ませれば大丈夫」のような断定は書かない
- [ ] 「個人差があります」「薬剤師にご相談ください」を必ず添える
- [ ] 出典（NCCHD資料）を明記する
- [ ] 「公的資料ベース」と表記する（「薬剤師監修」は実際に監修を受けていない限り使用不可）
- [ ] 他サイトの文章をコピーしない（Googleペナルティ＋著作権の問題）
- [ ] アプリへの誘導は自然に。宣伝臭を出さない

## 1. 戦略

### 狙うキーワード

「薬の商品名 + 子供 + 飲ませ方」がメインキーワード。
親が検索するのは「今まさに子どもが薬を飲んでくれない」瞬間。即答性が命。

### 記事の優先順位（検索ニーズが高い順）

| # | キーワード | 現在の1位 | 狙う理由 |
|---|-----------|----------|---------|
| 1 | クラリスドライシロップ 飲ませ方 | ミナカラ | 最も苦い＝最も困る。1位は一覧表なし。勝てる |
| 2 | ムコダイン 粉薬 飲ませ方 | QLife | 薬DBサイト。食品相性の情報なし。勝てる |
| 3 | カルボシステイン 子供 飲み方 | EPARKくすりの窓口 | ムコダインの一般名。両方で検索される |
| 4 | アジスロマイシン 子供 飲ませ方 | ミナカラ | ジスロマックの一般名。苦味が強い |
| 5 | タミフル 粉薬 飲ませ方 | 育児ブログ | インフルシーズンに急増。1位は根拠薄い。勝てる |
| 6 | ワイドシリン 飲ませ方 | クリニックブログ | よく処方される抗生剤。甘い薬 |
| 7 | 粉薬 子供 飲ませ方 まとめ | PMDA | 政府サイト。薬別相性なし。一覧表で差別化可能 |

### 競合との差別化

市場分析（2026-03-18実施）の結果、**粉薬×食品の味の相性を検索できるアプリは日本にも世界にも存在しない**ことが確認された。

| 競合（具体名） | 検索順位の実績 | 弱点 |
|---------------|-------------|------|
| **ミナカラ**（minacolor.com） | 「クラリス 飲ませ方」で1-2位独占 | 飲み合わせ一覧表がない。記事は正確だがスキャンしにくい |
| **PMDA**（pmda.go.jp） | 「粉薬 子供 飲ませ方」で1位 | 政府サイト。薬別の具体的な食品相性がない |
| **キャップスクリニック** | 複数キーワードで上位 | 薬別の相性表あり（最大の脅威）。ただし静的記事で検索・比較不可 |
| **QLife** | 「ムコダイン 飲み合わせ」で1位 | 薬データベースだが味×食品の相性は対象外 |
| **育児ブログ各種** | 「タミフル 飲ませ方」で1位 | 体験談ベース。根拠が薄い。情報が古い |
| **NCCHD PDF** | 各キーワードで10位前後 | 事実上の業界標準データだが、PDFで検索不可・スマホで読めない |

**おくすり味ナビ記事の強み（= 上記すべてに対する差別化）:**
- **唯一のアプリ**: 薬名で検索 → おすすめ/NGが即座にわかるサービスは他に存在しない
- **一覧表の即答性**: 全記事に飲み合わせ一覧表を掲載。ミナカラにもキャップスにもない形式
- **複数薬まとめ機能**: 処方箋に複数の薬がある場合、共通して使える食品を提示。競合には不可能
- **公的資料ベースの信頼性**: NCCHDのデータを正確に構造化。育児ブログの体験談とは次元が違う

## 2. 記事の公開場所

noteで公開する。

- 無料で始められる
- noteドメインは検索エンジンからの評価が比較的高い
- アプリへのリンクを自然に貼れる
- 開発秘話やアプリ紹介の記事も同じアカウントで書ける

記事数が増え、検索流入が安定してきた段階で独自ドメイン移行を検討する。

## 3. 記事テンプレート

薬は大きく2タイプに分かれる。テンプレートを使い分けること。

### 薬の味タイプ別テンプレート

| タイプ | 該当する薬（例） | 記事の切り口 |
|--------|-----------------|-------------|
| **苦い薬** | クラリス、ジスロマック、タミフル | 「苦くて飲めない → 正しい食品を選べば飲める」 |
| **甘い・飲みやすい薬** | ワイドシリン、ムコダイン | 「比較的飲みやすい薬。でも混ぜるものによっては味が変わる」 |

---

### タイトルの付け方

```
【公的資料ベース】〇〇（薬名）の飲ませ方｜おすすめ食品とNG食品まとめ
```

ポイント:
- 薬名を先頭に（検索ヒットのため）
- 「おすすめ」「NG」で即答感を出す
- 「公的資料ベース」で信頼性を示す

---

### 本文の構成

#### 冒頭（悩みに共感 → 即答）

**苦い薬の場合:**
```
子どもに〇〇（薬名）を処方されたけど、苦くて飲んでくれない…。
そんなときに試してほしい食品と、絶対に避けるべき食品をまとめました。

✅ おすすめ: △△、□□、▲▲
⚠ 避けるべき: ××、◇◇、●●

（※おすすめ/NGはアプリのデータから転記すること）
```

**甘い薬の場合:**
```
〇〇（薬名）は甘い味がついていて、粉薬の中では比較的飲みやすい薬です。
でも、混ぜるものによっては味が変わることも。相性の良い食品・悪い食品をまとめました。

✅ おすすめ: △△、□□、▲▲
▲ 注意: ××、◇◇

（※おすすめ/NGはアプリのデータから転記すること）
```

冒頭3行で答えを出す。「詳しくは下へ」は離脱の原因。

---

#### 飲み合わせ一覧表

**全記事に必ず入れる。これが最大の差別化要素。**

```
| 食品 | 相性 | ひとこと |
|------|:----:|---------|
| 🍫 チョコアイス | ◎ | （アプリデータから転記） |
| ☕ ココア | ◎ | ... |
| 🍮 プリン | △ | ... |
| ... | ... | ... |
```

NCCHD公式10食品（ココア、プリン、牛乳、アイス、お茶、ヨーグルト、りんごジュース、オレンジ飲料、乳酸菌飲料、スポーツ飲料）は必ず網羅する。アプリ独自のおすすめ食品（練乳など）がある場合は追加してもよい。

---

#### おすすめ食品の飲ませ方のコツ

一覧表で◎の食品ごとに、具体的な飲ませ方のポイントを簡潔に記載する。
読者は「何がおすすめか」を知った直後に「どう使うか」を知りたい。

---

#### 飲ませ方の鉄則（全記事共通で使い回し可）

```
① 飲む直前に混ぜる（放置すると苦味が出る）
② 「混ぜる」より「挟む」（アイスやプリンは中に埋め込む）
③ 飲んだ後に水で口をすすぐ
④ ミルクやごはんには混ぜない（主食が嫌いになる恐れ）
⑤ 飲めなくても叱らない
```

---

#### 避けるべき食品の理由

苦い薬の場合: 「酸性食品がNGな理由（コーティング溶解の仕組み）」を説明すると説得力が増す。

---

#### 他の薬と一緒に処方された場合

```
複数の薬をまとめて飲み合わせを確認したい場合は、
「おくすり味ナビ」の「複数薬まとめ」機能が便利です。
→ [アプリへのリンク]
```

---

#### 薬の基本情報（表形式）

```
| 項目 | 内容 |
|------|------|
| 商品名 | 〇〇 |
| 一般名 | △△ |
| 製造元 | □□ |
| 味 | ×× |
| 主な用途 | ... |
| 主なジェネリック | ... |
```

---

#### 末尾: 出典＋アプリ紹介（URLなし）

アプリへのリンク（CTA）は記事中盤の「複数薬まとめ機能」紹介箇所に1箇所だけ配置する。
末尾にもURLを入れると宣伝感が出るため、アプリ名の言及のみにとどめる。

```
---
出典: 国立成育医療研究センター薬剤部
「粉薬と服薬補助食品の飲み合わせ」（2018年）
https://www.ncchd.go.jp/hospital/about/section/medicine/nomiawase.pdf

本記事は医療行為の代替ではありません。
服薬に関する最終判断は医師・薬剤師にご確認ください。
---

📱 おくすり味ナビ — 公的資料ベースで粉薬の味・飲み合わせを確認できるアプリです。
スマホのホーム画面に追加するとアプリとして使えます。
```

## 4. 総合記事テンプレート（薬名を特定しない検索向け）

### タイトル例

```
【保存版】子どもの粉薬が飲めない時の飲ませ方ガイド｜薬別おすすめ＆NG食品一覧
```

### 構成

1. 冒頭: 「粉薬が飲めない」は多くの親の悩み → この記事で一覧にまとめた
2. 粉薬の飲ませ方の基本（食品に混ぜる方法、服薬ゼリー）
3. **薬ごとのおすすめ＆NG一覧表**（主要5〜8薬をまとめた横断表。これが記事の核）
4. やってはいけない3つのこと（主食に混ぜない、酸性飲料に注意、放置しない）
5. 複数薬が処方された時の注意点 → アプリの「まとめ機能」へ誘導
6. 出典＋アプリ紹介

### 横断表の例

```
| 薬名 | チョコアイス | ヨーグルト | スポーツ飲料 | ... |
|------|:----------:|:---------:|:----------:|-----|
| クラリス | ◎ | ✕ | ✕ | ... |
| ムコダイン | ◎ | ◎ | ◎ | ... |
| タミフル | ◎ | ◎ | ◎ | ... |
| ... | ... | ... | ... | ... |
```

※この表はアプリのデータから生成すること。絶対に推測で埋めない。

## 5. 記事作成チェックリスト

記事を書くたびにこのチェックリストを使う。

### 執筆前
- [ ] アプリで対象の薬を開き、公式10食品の相性を確認した
- [ ] mixGood / mixOk / mixNG のデータも確認した
- [ ] ジェネリック医薬品の情報を確認した

### 執筆中
- [ ] タイトルに薬名が入っている
- [ ] 冒頭3行で結論（おすすめ/NG）を出している
- [ ] 一覧表を入れた
- [ ] 飲ませ方の鉄則を入れた
- [ ] アプリへの導線を自然に入れた（複数薬まとめ機能の箇所に1箇所のみ）
- [ ] 出典を明記した

### 公開前
- [ ] おすすめ/NGの全ペアがアプリのデータと一致することを最終確認した
- [ ] 「断定表現」がないか読み返した
- [ ] 「個人差があります」等の注意書きが入っている

### 公開後
- [ ] noteの記事URLをXで投稿した
- [ ] Google Search Console でインデックス登録をリクエストした

## 6. 執筆スケジュールの目安

- 1記事あたり1,500〜2,500文字（長すぎると書く負担が重い）
- 週1〜2本ペースで無理なく続ける
- 最初の1本はクラリスドライシロップ（最も検索ニーズが高い）
- 5本書いた時点で検索流入の反応を見て、残りの優先順位を調整
