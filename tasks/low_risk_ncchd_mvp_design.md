# NCCHDベース低リスク版 設計書

## 1. 目的
- 現行の「こなぐすり味ナビ」を、`AIの推測で答えるアプリ` ではなく `公的機関の公開資料を素早く引けるナビ` に寄せる
- 初期公開時点では、国立成育医療研究センターの公開資料にある客観情報だけを表示し、未掲載薬は無理に埋めない
- デザインの雰囲気は現状維持とし、情報構造と公開ルールだけを低リスク化する

## 2. 参照ソース
- 国立成育医療研究センター「お薬の飲ませ方について」
  - https://www.ncchd.go.jp/hospital/about/section/medicine/210330.html
- 同ページから配布されている PDF
  - https://www.ncchd.go.jp/hospital/about/section/medicine/nomiawase.pdf

## 3. 前提整理
- このアプリが扱う中心データは、上記 PDF の `粉薬の味・香り・服薬補助食品との相性` とする
- 初期公開では、PDF に掲載されている薬だけを公開対象にする
- PDF にない薬は `未掲載` として扱い、AI やネット検索だけで公開文言を補完しない
- メーカー差の注記があるため、`同成分なら同じ` とみなして自動横展開しない
- `薬剤師監修` の表示は、別途レビュー記録が揃うまで使わない

## 4. プロダクト方針
### 4-1. 公開時の見せ方
- アプリ名や本文では `国立成育医療研究センター公開資料をもとに整理` と表現する
- `おすすめ / まあまあ / NG` は廃止し、資料準拠の `飲みやすい / 変わらない / 飲みにくい / データなし` に統一する
- 味や香りは、資料にある範囲だけを表示する
- 注意文は総論として固定表示する
  - 混ぜたらすぐ飲む
  - 酸性飲料で苦味や含量低下の可能性がある
  - 味や香りはメーカーで異なることがある

### 4-2. 公開時にやらないこと
- 未掲載薬への推測回答
- ジェネリックや別メーカーへの自動適用
- `この子にはこれが最適` のような個別最適化
- 症状から薬を選ばせる診断的な導線
- AIが生成した混ぜ方コメントの公開表示

## 5. データモデル
公開用の薬データは、現行の自由記述中心の構造から次の2層に分ける。

### 5-1. `officialRecord`
- `id`
- `displayName`
- `genericName`
- `makerLabel`
- `form`
- `categories`
- `tasteLabel`
- `flavorLabel`
- `compatibility`
  - `cocoa`
  - `iceCream`
  - `milk`
  - `yogurt`
  - `tea`
  - `juice`
  - `appleJuice`
  - `orangeJuice`
  - `sportsDrink`
  - `warmWater`
- `source`
  - `publisher`
  - `title`
  - `page`
  - `url`
  - `publishedNote`

### 5-2. `internalMeta`
- `listedInOfficialSource`
- `sourceVersion`
- `verifiedImportedBy`
- `verifiedImportedOn`
- `notesInternal`
- `review`
  - 既存の `verified / verifiedOn / reviewer / scope / sourceRefs / note`

## 6. データ投入ルール
### 6-1. 公開可
- PDF に明示的に載っている薬
- PDF の評価記号をそのまま構造化したもの
- 画面文言は資料の意味を変えない短い整理にとどめる

### 6-2. 非公開
- PDF に載っていない薬
- PDF に載っていても製品名が曖昧なもの
- 味や相性を別資料から寄せ集めたもの
- 担当者が一次資料照合を終えていないもの

### 6-3. 保留
- 別の公的資料や製薬会社資料にはあるが、NCCHD PDF にはない薬
- こうした薬は `候補データ` として内部保存だけし、公開しない

## 7. UI仕様
### 7-1. 一覧ページ
- 現在のカードUIを維持
- カード上部に `出典: 国立成育医療研究センター公開資料` バッジを追加
- 閉じた状態では、上位2候補ではなく `資料上の評価記号` を優先表示する
  - 例: `飲みやすい: ココア, アイス`
  - `飲みにくい: オレンジジュース`
- 未掲載薬は検索結果に出さない
- 検索ヒットなし時は `公開資料掲載の薬で一致するものがありません` と表示

### 7-2. 飲み合わせ表
- 現在の表UIを維持
- 凡例を `飲みやすい / 変わらない / 飲みにくい / データなし` に統一
- 表の上に総論注意を固定表示
- 各セルの意味は PDF 準拠とし、独自評価は混ぜない

### 7-3. 詳細カード
- 現在の詳細構成は維持
- `味・後味` の自由記述は廃止し、まずは `味` `香り` `出典` だけを見せる
- `混ぜるとよいもの` という表現は `資料上の相性` に変更する
- 自由文の `caution` は総論または出典付き注意文だけに絞る

### 7-4. 出典表示
- 各カードに出典リンクを付与
- 詳細内に `最終確認日` と `ソース版` を表示
- フッターに `本アプリは公開資料を検索しやすく整理したもの` と明記

## 8. 検索仕様
- 検索対象は `製品名 / 一般名 / カテゴリ / メーカー表記`
- AI の役割はローカルな入力補助だけにとどめる
  - ひらがな・カタカナ揺れ
  - 全角半角の吸収
  - 代表的な別名辞書
- AI が新しい飲み合わせ評価を生成して返す機能は持たない

## 9. 公開文言ガイド
### 9-1. 推奨文言
- `国立成育医療研究センターの公開資料をもとに、粉薬・漢方薬の相性情報を見やすく整理しています`
- `掲載内容は公開資料に基づくもので、未掲載薬は表示していません`

### 9-2. 避ける文言
- `専門家監修`
- `この組み合わせが最適`
- `必ず飲める`
- `全薬対応`
- `AIが判断`

## 10. 運用フロー
### 10-1. 初期リリース
- NCCHD PDF 掲載薬のみ公開
- 既存データのうち、NCCHD 非掲載薬は内部候補へ退避
- 監修ページは内部運用のまま残す

### 10-2. 更新時（データ更新方針）

市場分析（2026-03-18）で元データの古さ（2004-2007年の論文ベース）がWeaknessとして特定された。
以下のルールでデータの鮮度を維持する。

**定期チェック（年1回、毎年4月）:**
- NCCHD飲み合わせPDF（https://www.ncchd.go.jp/hospital/about/section/medicine/nomiawase.pdf）の更新有無を確認
- NCCHDのHTMLページ（https://www.ncchd.go.jp/hospital/about/section/medicine/210330.html）も合わせて確認
- 補完候補として以下の病院資料も定期チェック:
  - 神奈川県立こども医療センター薬剤部の飲み合わせ資料
  - 北信総合病院の院内資料

**更新があった場合:**
- PDF 差分を手動確認
- 更新した薬だけ `sourceVersion` と `verifiedImportedOn` を更新
- 変更履歴を `tasks/` に残す

**アプリ内の鮮度表示:**
- 各カードに出典の年次を表示する（「出典: NCCHD 2018年資料」）
- フッターに「データ最終確認日: YYYY-MM-DD」を表示する

### 10-3. 将来拡張
- 彼女さんなど薬剤師レビュー済みの追加薬だけ、`reviewedExtension` レイヤーとして別管理する
- UI 上は `公的資料掲載` と `薬剤師確認済み追加` を明確に分ける
- 同じカード内でソース種別を混ぜない

## 11. 実装方針
### 11-1. 既存HTMLへの最小変更案
- 現在の `medicines` を次の3配列へ分ける
  - `officialMedicines`
  - `reviewedExtensions`
  - `internalCandidates`
- 公開画面は初期状態で `officialMedicines` のみ参照
- 既存の `review` 仕組みは維持しつつ、`officialSource` フラグを追加

### 11-2. 副作用リスク
- 既存の自由文を削るため、情報量は一時的に減る
- 既存の薬数は大幅に減る可能性がある
- ただし、公開範囲が狭くなることで誤記載リスクは下げやすい

## 12. MVP完了条件
- 公開画面に NCCHD 掲載薬以外が出ない
- すべての公開薬に出典情報がある
- 表示評価が `飲みやすい / 変わらない / 飲みにくい / データなし` に統一されている
- 未掲載薬へ推測回答しない
- ストア説明文に `AI判断` や `監修` と誤解される文言がない
- アプリ内に出典の年次が表示されている（データの鮮度を透明化）

## 13. 次の実装タスク
1. 現在の全薬を `NCCHD掲載 / 非掲載 / 保留` に棚卸しする
2. 公開用の評価ラベルを PDF 準拠へ変更する
3. 自由記述の味・混ぜ方文言を出典中心表示へ差し替える
4. 出典バッジと出典リンクをカードに追加する
5. 未掲載薬を内部候補へ退避し、公開検索から除外する
